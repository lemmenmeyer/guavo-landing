# guavo-landing

Static landing site for guavo.com plus two Vercel serverless functions.

## API endpoints

- `POST /api/submit-application` — receives the app-form submission from the
  client, sends broker + applicant copies via Resend, and creates the fully-
  populated Monday item on the Guavo Pipeline board with the 3 attachments
  routed to their file columns (App form PDF, ID Document, Bank Statements).
- `POST /api/send-decline` — triggered by a Monday "When Decline Email Status
  changes to Ready, send webhook" automation on the Guavo Pipeline board
  (id 18416816603). Composes a compliant adverse-action notice via the
  reason-specific template, then **creates a Gmail draft** in
  daniel@guavo.com's mailbox (never auto-sends). Daniel reviews, edits if
  needed, sends manually, then flips Monday status to Sent.

## Environment variables

Set these in Vercel Project Settings → Environment Variables:

| Variable | Used by | Notes |
|---|---|---|
| `RESEND_API_KEY` | submit-application | `re_...` key from resend.com/api-keys. |
| `RESEND_FROM` | submit-application | Optional. Defaults to `Guavo Applications <contact@guavo.com>`. |
| `RESEND_TO` | submit-application | Optional. Defaults to `apply@guavo.com`. |
| `MONDAY_API_TOKEN` | both | Personal API v2 token from staura.monday.com Developer Center. |
| `MONDAY_WEBHOOK_SECRET` | send-decline | Long random string. Appended to the Monday automation's webhook URL as `?secret=...` OR sent as `X-Guavo-Webhook-Secret` header. |
| `GMAIL_CLIENT_ID` | send-decline | Google Cloud OAuth 2.0 Client ID (Desktop app type). |
| `GMAIL_CLIENT_SECRET` | send-decline | Paired with the Client ID. |
| `GMAIL_REFRESH_TOKEN` | send-decline | Generated by `~/.guavo-underwriting/intake/gmail_auth.mjs --email daniel@guavo.com` with `gmail.modify` scope. |

## Decline templates

`lib/decline-templates/` — one file per Decline Reason label on Monday plus a
shared `common.js` module holding the FCRA §615(a) score-disclosure block and
the visual shell. `index.js` maps Monday label text → template module.

Currently mapped labels: `Low FICO`, `Low ADB`, `Under Minimum Revenue`,
`Location`. Any Decline Reason label not in `index.js` no-ops with an item
Update explaining what's missing.

## Local template preview

Dry-run renderer at `~/.guavo-underwriting/dev/send_decline_dryrun.mjs`
imports the same template modules and writes HTML + plain-text previews to a
temp directory — useful for reviewing wording changes before deploying. Never
calls Gmail, never mutates Monday.

## Monday board dependencies

Column ids consumed by `/api/send-decline` are pinned in the file's `COL`
object. If any of those columns are renamed or replaced on Monday, update
the ids there and in `guavo_monday_workspace.md` memory.
